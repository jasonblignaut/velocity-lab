FortiGate Routing Configuration

Last updated: July 17, 2025

This comprehensive guide covers configuring static and dynamic routing on FortiGate devices to ensure proper network connectivity and optimal traffic flow.

Important

Routing changes can affect network connectivity. Always test routing configurations in a controlled environment before implementing in production.

Overview

FortiGate routing functionality includes static routes, dynamic routing protocols (OSPF, BGP, RIP), policy-based routing, and route maps. Understanding these components is essential for designing efficient and resilient network architectures.

Routing Components

  • Static Routes: Manually configured routes for specific destinations
  • Dynamic Protocols: OSPF, BGP, RIP for automatic route discovery
  • Policy Routes: Route traffic based on source, destination, or service
  • Route Maps: Control route redistribution and modification
  • Administrative Distance: Priority for route selection

Prerequisites

  • FortiGate device with configured interfaces
  • Network topology documentation
  • IP addressing scheme planned
  • Understanding of routing protocols and concepts
  • Access to neighboring routers for dynamic routing

CLI Static Route Configuration

  1. Create basic static route 
    config router static
edit 1
    set dst 192.168.100.0 255.255.255.0
    set gateway 192.168.1.1
    set device "port2"
    set distance 10
    set priority 5
    set comment "Route to Branch Office"
next
end
  2. Create default route 
    config router static
edit 2
    set dst 0.0.0.0 0.0.0.0
    set gateway 203.0.113.1
    set device "port1"
    set distance 1
    set comment "Default route to Internet"
next
end
  3. Create load-balanced routes 
    config router static
edit 3
    set dst 10.0.0.0 255.0.0.0
    set gateway 192.168.1.10
    set device "port3"
    set distance 15
    set priority 10
next
edit 4
    set dst 10.0.0.0 255.0.0.0
    set gateway 192.168.1.11
    set device "port4"
    set distance 15
    set priority 10
next
end
  4. Configure route with link monitoring 
    config router static
edit 5
    set dst 172.16.0.0 255.255.0.0
    set gateway 192.168.1.20
    set device "port5"
    set link-monitor-exempt disable
    set comment "Monitored route"
next
end

Dynamic routing protocols automatically discover and maintain routing information between routers.

    Parameter Description Example
    Router ID Unique OSPF identifier 1.1.1.1
    Area ID OSPF area identifier 0.0.0.0
    Network Type OSPF network type Broadcast
    Hello Interval Hello packet interval 10 seconds
    Dead Interval Dead timer interval 40 seconds
  4. Configure OSPF interfaces
    Screenshot: OSPF interface configuration
  5. Configure area settings
    Screenshot: OSPF area configuration

CLI Dynamic Routing Configuration

  1. Configure OSPF 
    config router ospf
    set router-id 1.1.1.1
    set auto-cost-ref-bandwidth 100000
    config area
        edit 0.0.0.0
            set type regular
            set shortcut default
        next
    end
    config ospf-interface
        edit "internal"
            set interface "internal"
            set cost 10
            set hello-interval 10
            set dead-interval 40
            set priority 1
        next
        edit "dmz"
            set interface "dmz"
            set cost 20
            set hello-interval 10
            set dead-interval 40
        next
    end
    config network
        edit 1
            set prefix 192.168.1.0 255.255.255.0
            set area 0.0.0.0
        next
        edit 2
            set prefix 192.168.2.0 255.255.255.0
            set area 0.0.0.0
        next
    end
end
  2. Configure BGP 
    config router bgp
    set as 65001
    set router-id 1.1.1.1
    set keepalive-timer 60
    set holdtime-timer 180
    config neighbor
        edit "203.0.113.1"
            set remote-as 65000
            set update-source "port1"
            set route-map-out "BGP_OUT"
            set route-map-in "BGP_IN"
        next
    end
    config network
        edit 1
            set prefix 192.168.1.0 255.255.255.0
        next
    end
end
  3. Configure RIP 
    config router rip
    set default-information-originate enable
    set default-metric 5
    set max-out-metric 15
    set timeout-timer 180
    set update-timer 30
    config interface
        edit "internal"
            set auth-mode text
            set auth-string "ripkey123"
            set send-version 2
            set receive-version 2
        next
    end
    config network
        edit 1
            set prefix 192.168.1.0 255.255.255.0
        next
    end
end

Policy Routes

Policy-based routing allows you to route traffic based on criteria other than destination IP address.

Warning

Policy routes are processed before normal routing table lookups. Incorrect policy routes can cause traffic to be misrouted.

Policy Route Configuration via GUI

  1. Navigate to Network > Policy Routes
    Screenshot: Policy Routes page
  2. Click "Create New" to add policy route
    Screenshot: Create policy route form
  3. Configure policy route criteria
    Screenshot: Policy route criteria configuration
    Criteria Description Example
    Source Source IP or subnet 192.168.10.0/24
    Destination Destination IP or subnet 0.0.0.0/0
    Protocol IP protocol TCP
    Service Service or port HTTP
    Input Interface Incoming interface internal
  4. Configure routing action
    Screenshot: Policy route action configuration
  5. Set priority and enable status
    Screenshot: Policy route priority settings

CLI Policy Route Configuration

  1. Create policy route for specific traffic 
    config router policy
edit 1
    set input-device "internal"
    set src "192.168.10.0/24"
    set dst "0.0.0.0/0"
    set protocol 6
    set start-port 80
    set end-port 80
    set gateway 192.168.1.10
    set output-device "port2"
    set comments "Route web traffic via backup link"
next
end
  2. Create policy route based on source 
    config router policy
edit 2
    set input-device "internal"
    set src "192.168.20.0/24"
    set dst "0.0.0.0/0"
    set gateway 192.168.1.20
    set output-device "port3"
    set comments "Route marketing subnet via dedicated link"
next
end
  3. Create policy route with TOS matching 
    config router policy
edit 3
    set input-device "internal"
    set src "192.168.1.0/24"
    set dst "0.0.0.0/0"
    set tos 0x10
    set tos-mask 0xFC
    set gateway 192.168.1.5
    set output-device "port4"
    set comments "Route high priority traffic"
next
end

CLI Route Map Configuration

  1. Create route map for BGP 
    config router route-map
edit "BGP_OUT"
    config rule
        edit 1
            set action permit
            set match-ip-address "INTERNAL_NETWORKS"
            set set-metric 100
            set set-local-preference 200
        next
        edit 2
            set action deny
        next
    end
next
end
  2. Create access list for route map 
    config router access-list
edit "INTERNAL_NETWORKS"
    config rule
        edit 1
            set action permit
            set prefix 192.168.1.0 255.255.255.0
        next
        edit 2
            set action permit
            set prefix 192.168.2.0 255.255.255.0
        next
        edit 3
            set action deny
            set prefix 0.0.0.0 0.0.0.0
        next
    end
next
end
  3. Create prefix list for route filtering 
    config router prefix-list
edit "ALLOWED_PREFIXES"
    config rule
        edit 1
            set action permit
            set prefix 192.168.0.0 255.255.0.0
            set ge 24
            set le 30
        next
        edit 2
            set action deny
            set prefix 0.0.0.0 0.0.0.0
            set le 32
        next
    end
next
end

Multicast Routing

Configure multicast routing for applications that require one-to-many communication.

    Screenshot: PIM configuration

    Screenshot: IGMP configuration

    Screenshot: Multicast policy configuration

CLI Multicast Configuration

  1. Enable multicast routing 
    config router multicast
    set multicast-routing enable
    config interface
        edit "internal"
            set pim-mode sparse-mode
            set igmp-version 3
            set join-group "224.0.0.0/4"
        next
        edit "dmz"
            set pim-mode sparse-mode
            set igmp-version 3
        next
    end
end
  2. Configure PIM settings 
    config router multicast
    config pim-sm-global
        set message-interval 60
        set join-prune-holdtime 210
        set accept-register-checksum enable
        set accept-source-checksum enable
        set bsr-candidate enable
        set bsr-interface "internal"
        set bsr-priority 10
        set bsr-hash 10
        set rp-candidate enable
        set rp-candidate-interface "internal"
        set rp-candidate-priority 10
        set rp-candidate-interval 60
    end
end
  3. Configure multicast flow control 
    config router multicast-flow
edit "VIDEO_STREAM"
    set group-addr 239.1.1.1 255.255.255.255
    set source-addr 192.168.1.100 255.255.255.255
    set comments "Video streaming multicast"
next
end
