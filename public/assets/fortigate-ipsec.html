FortiGate IPSec VPN Setup

Last updated: July 17, 2025

This comprehensive guide covers configuring IPSec VPN tunnels on FortiGate devices for secure site-to-site connectivity between networks.

Important

IPSec VPN requires compatible encryption algorithms and proper firewall policies. Ensure both endpoints can reach each other's public IP addresses.

Overview

IPSec (Internet Protocol Security) VPN creates secure encrypted tunnels between FortiGate devices over the internet. This allows branch offices to securely connect to headquarters or connect multiple sites in a hub-and-spoke topology.

IPSec Components

  • Phase 1 (IKE): Establishes secure channel and authenticates peers
  • Phase 2 (IPSec): Negotiates encryption parameters for data transfer
  • Tunnels: Virtual interfaces that carry encrypted traffic
  • Policies: Firewall rules controlling traffic flow through tunnels

Prerequisites

  • Two FortiGate devices with public IP addresses
  • Administrative access to both FortiGate devices
  • Non-overlapping private IP subnets at each site
  • Pre-shared key or certificate for authentication
  • Knowledge of remote site's public IP and internal subnets

Phase 1 Setup

Phase 1 establishes the secure channel between the two FortiGate devices and handles authentication.

Phase 1 Configuration via GUI

    Name Tunnel identifier Branch_Office_VPN
    Remote Gateway Static IP Address 203.0.113.50
    Interface Local WAN interface port1
    Pre-shared Key Authentication key SecureKey123!
  4. Configure advanced Phase 1 settings
    Screenshot: Advanced Phase 1 settings
  5. Set encryption and authentication algorithms
    Screenshot: Encryption algorithm selection

CLI Phase 1 Configuration

  1. Configure Phase 1 interface 
    config vpn ipsec phase1-interface
edit "Branch_Office_VPN"
    set interface "port1"
    set keylife 86400
    set peertype any
    set net-device disable
    set proposal aes256-sha256 aes256-sha1 aes128-sha256 aes128-sha1
    set dhgrp 14 5
    set remote-gw 203.0.113.50
    set psksecret "SecureKey123!"
    set dpd-retryinterval 60
next
end
  2. Configure Dead Peer Detection (DPD) 
    config vpn ipsec phase1-interface
edit "Branch_Office_VPN"
    set dpd enable
    set dpd-retrycount 3
    set dpd-retryinterval 20
next
end
  3. Configure NAT traversal if needed 
    config vpn ipsec phase1-interface
edit "Branch_Office_VPN"
    set nattraversal enable
    set fragmentation enable
next
end

Phase 2 Setup

Phase 2 defines the encryption parameters for the actual data transfer and specifies which traffic should be encrypted.

Phase 2 Configuration via GUI

  1. In the same IPSec tunnel dialog, configure Phase 2
    Screenshot: Phase 2 configuration tab
  2. Set local and remote subnets
    Screenshot: Local and remote subnet configuration
    Setting Description Example
    Local Subnet Local network range 192.168.1.0/24
    Remote Subnet Remote network range 192.168.100.0/24
    Encryption Data encryption algorithm AES-256
    Authentication Data authentication SHA-256
  3. Configure PFS (Perfect Forward Secrecy)
    Screenshot: PFS configuration options
  4. Set key lifetime and replay protection
    Screenshot: Key lifetime and replay settings

CLI Phase 2 Configuration

  1. Configure Phase 2 interface 
    config vpn ipsec phase2-interface
edit "Branch_Office_VPN"
    set phase1name "Branch_Office_VPN"
    set proposal aes256-sha256 aes256-sha1 aes128-sha256 aes128-sha1
    set pfs enable
    set dhgrp 14 5
    set replay enable
    set keylifeseconds 43200
    set src-subnet 192.168.1.0 255.255.255.0
    set dst-subnet 192.168.100.0 255.255.255.0
next
end
  2. Configure multiple subnets if needed 
    config vpn ipsec phase2-interface
edit "Branch_Office_VPN_DMZ"
    set phase1name "Branch_Office_VPN"
    set proposal aes256-sha256
    set pfs enable
    set dhgrp 14
    set src-subnet 192.168.2.0 255.255.255.0
    set dst-subnet 192.168.101.0 255.255.255.0
next
end
  3. Enable auto-negotiate 
    config vpn ipsec phase2-interface
edit "Branch_Office_VPN"
    set auto-negotiate enable
next
end

Static Routes

Configure static routes to direct traffic through the IPSec tunnel to reach remote subnets.

Static Route Configuration via GUI

  1. Navigate to Network > Static Routes
    Screenshot: Static Routes page
  2. Click "Create New" to add route
    Screenshot: Create new static route form
  3. Configure route to remote subnet
    Field Value Description
    Destination 192.168.100.0/24 Remote subnet
    Gateway 0.0.0.0 Not applicable for VPN
    Interface Branch_Office_VPN VPN tunnel interface
    Administrative Distance 10 Route priority
  4. Verify route configuration
    Screenshot: Static route verification

CLI Static Route Configuration

  1. Add static route for remote subnet 
    config router static
edit 0
    set dst 192.168.100.0 255.255.255.0
    set device "Branch_Office_VPN"
    set comment "Route to branch office LAN"
next
end
  2. Add route for additional remote subnets 
    config router static
edit 0
    set dst 192.168.101.0 255.255.255.0
    set device "Branch_Office_VPN"
    set comment "Route to branch office DMZ"
next
end
  3. Verify routing table 
    get router info routing-table static

Firewall Policies

Create firewall policies to allow traffic through the IPSec tunnel between local and remote networks.

Warning

Without proper firewall policies, traffic will not flow through the IPSec tunnel even if it's established successfully.

Firewall Policy Configuration via GUI

  1. Navigate to Policy & Objects > Firewall Policy
    Screenshot: Firewall Policy page
  2. Create policy for local to remote traffic
    Screenshot: Policy creation form
    Field Value Description
    Name Local_to_Branch Policy identifier
    Incoming Interface internal Local LAN interface
    Outgoing Interface Branch_Office_VPN VPN tunnel interface
    Source 192.168.1.0/24 Local subnet
    Destination 192.168.100.0/24 Remote subnet
    Service ALL All services
    Action ACCEPT Allow traffic
  3. Create policy for remote to local traffic
    Screenshot: Reverse policy configuration
  4. Enable logging for troubleshooting
    Screenshot: Logging configuration

CLI Firewall Policy Configuration

  1. Create policy for local to remote traffic 
    config firewall policy
edit 0
    set name "Local_to_Branch"
    set srcintf "internal"
    set dstintf "Branch_Office_VPN"
    set srcaddr "192.168.1.0/24"
    set dstaddr "192.168.100.0/24"
    set action accept
    set schedule "always"
    set service "ALL"
    set logtraffic all
next
end
  2. Create policy for remote to local traffic 
    config firewall policy
edit 0
    set name "Branch_to_Local"
    set srcintf "Branch_Office_VPN"
    set dstintf "internal"
    set srcaddr "192.168.100.0/24"
    set dstaddr "192.168.1.0/24"
    set action accept
    set schedule "always"
    set service "ALL"
    set logtraffic all
next
end
  3. Configure address objects for easier management 
    config firewall address
edit "Local_LAN"
    set subnet 192.168.1.0 255.255.255.0
    set comment "Local office LAN"
next
edit "Branch_LAN"
    set subnet 192.168.100.0 255.255.255.0
    set comment "Branch office LAN"
next
end

Troubleshooting

Common IPSec VPN issues and their solutions.

Common Issues and Solutions

  1. Phase 1 not establishing
    Screenshot: Phase 1 failure indicators
    • Verify pre-shared key matches on both sides
    • Check that remote gateway IP is correct
    • Ensure encryption proposals match
    • Verify UDP ports 500 and 4500 are open
  2. Phase 2 not establishing
    Screenshot: Phase 2 failure indicators
    • Check that local and remote subnets are correctly configured
    • Verify encryption and authentication algorithms match
    • Ensure PFS settings are compatible
  3. Tunnel up but no traffic flow
    Screenshot: Traffic flow troubleshooting
    • Check firewall policies allow VPN traffic
    • Verify static routes are configured
    • Ensure no IP conflicts between sites

CLI Troubleshooting Commands

  1. Debug Phase 1 negotiations 
    diagnose debug reset
diagnose debug enable
diagnose debug application ike 255
  2. Debug Phase 2 negotiations 
    diagnose debug application ipsec 255
  3. Reset IPSec tunnel 
    diagnose vpn ike restart
diagnose vpn ipsec tunnel down Branch_Office_VPN
diagnose vpn ipsec tunnel up Branch_Office_VPN
  4. Check system resources 
    get system performance status
get system status
  5. Monitor packet flow 
    diagnose sniffer packet any 'host 192.168.100.1' 4 0 l
