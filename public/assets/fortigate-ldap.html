FortiGate LDAP Authentication

Last updated: July 17, 2025

This comprehensive guide covers configuring LDAP authentication on FortiGate devices to integrate with Active Directory or other LDAP servers for centralized user management.

Important

LDAP authentication requires network connectivity to the LDAP server and proper bind credentials. Test connectivity before implementing in production.

Overview

LDAP (Lightweight Directory Access Protocol) authentication allows FortiGate devices to authenticate users against centralized directory services like Active Directory, OpenLDAP, or other LDAP-compliant systems. This eliminates the need to maintain separate user accounts on each FortiGate device.

Benefits of LDAP Authentication

  • Centralized Management: Single source of truth for user accounts
  • Simplified Administration: Manage users in Active Directory
  • Enhanced Security: Leverage existing password policies
  • Group-based Access: Use AD groups for authorization
  • Audit Trail: Centralized logging of authentication events

Prerequisites

  • FortiGate device with network connectivity to LDAP server
  • Active Directory or LDAP server with user accounts
  • Service account with read access to LDAP directory
  • Knowledge of LDAP server configuration (IP, port, base DN)
  • Understanding of LDAP search filters and attributes

LDAP Server

Configure the LDAP server connection settings to establish communication with your directory service.

LDAP Server Configuration via GUI

  1. Navigate to User & Authentication > LDAP Servers
    Screenshot: LDAP Servers page
  2. Click "Create New" to add LDAP server
    Screenshot: Create new LDAP server form
  3. Configure server connection settings
    Screenshot: LDAP server connection settings
    Setting Description Example
    Name LDAP server identifier Company_AD
    Server IP/Name Domain controller address 192.168.1.10
    Port LDAP port (389 or 636 for SSL) 389
    Common Name Identifier Username attribute sAMAccountName
    Distinguished Name Base search DN DC=company,DC=com
    Bind Type Authentication method Regular
    Username Service account CN=svc_fortinet,OU=Service Accounts,DC=company,DC=com
  4. Configure security settings
    Screenshot: LDAP security options
  5. Test the connection
    Screenshot: Test connection button and results

CLI LDAP Server Configuration

  1. Create LDAP server configuration 
    config user ldap
edit "Company_AD"
    set server "192.168.1.10"
    set cnid "sAMAccountName"
    set dn "DC=company,DC=com"
    set type regular
    set username "CN=svc_fortinet,OU=Service Accounts,DC=company,DC=com"
    set password "ServiceAccountPassword123!"
    set group-member-check user-attr
    set group-search-base "DC=company,DC=com"
    set group-object-filter "(&(objectcategory=group)(member=*))"
    set member-attr "member"
    set secure disable
    set port 389
next
end
  2. Configure SSL/TLS encryption 
    config user ldap
edit "Company_AD"
    set secure ldaps
    set port 636
    set ca-cert "LDAP_CA_Cert"
next
end
  3. Test LDAP connectivity 
    diagnose test authserver ldap Company_AD john.doe SecurePassword123!
  4. Configure group search settings 
    config user ldap
edit "Company_AD"
    set group-search-base "OU=Groups,DC=company,DC=com"
    set group-object-filter "(&(objectCategory=group)(cn=*))"
    set group-member-check user-attr
    set member-attr "memberOf"
next
end

CLI User Group Configuration

  1. Create user group with LDAP mapping 
    config user group
edit "VPN_Users"
    set group-type firewall
    config match
        edit 1
            set server-name "Company_AD"

Authentication

Configure authentication policies and schemes to use LDAP for various services.

Authentication Policy Configuration

  1. Navigate to User & Authentication > Auth Schemes
    Screenshot: Authentication schemes page
  2. Create new authentication scheme
    Screenshot: Create auth scheme form
  3. Configure scheme settings
    Setting Description Example
    Name Scheme identifier LDAP_Auth
    Method Authentication method NTLM
    Negotiate NTLM Enable NTLM negotiation Enable
    Domain Controller LDAP server reference Company_AD
  4. Configure fallback authentication
    Screenshot: Fallback authentication settings

CLI Authentication Configuration

  1. Create authentication scheme 
    config authentication scheme
edit "LDAP_Auth"
    set method ntlm
    set negotiate-ntlm enable
    set domain-controller "Company_AD"
    set user-database "Company_AD"
next
end
  2. Configure authentication rule 
    config authentication rule
edit "LDAP_Rule"
    set srcintf "any"
    set srcaddr "all"
    set dstaddr "all"
    set protocol http https
    set auth-scheme "LDAP_Auth"
    set comments "LDAP authentication rule"
next
end
  3. Create identity-based policy 
    config firewall identity-based-policy
edit 1
    set schedule "always"
    set users "VPN_Users"
    config policy
        edit 1
            set srcintf "internal"
            set dstintf "any"
            set srcaddr "all"
            set dstaddr "all"
            set service "ALL"
            set action accept
        next
    end
next
end

SSL VPN with LDAP

Configure SSL VPN to use LDAP authentication for remote access users.

Warning

Ensure LDAP connectivity is stable before implementing SSL VPN with LDAP authentication to avoid locking out remote users.

SSL VPN LDAP Integration

  1. Navigate to VPN > SSL-VPN Settings
    Screenshot: SSL-VPN Settings page
  2. Configure authentication source
    Screenshot: Authentication source settings
  3. Assign user groups to SSL VPN portal
    Screenshot: User group assignment
  4. Configure group authentication settings
    Screenshot: Group authentication options
  5. Test SSL VPN with LDAP user
    Screenshot: SSL VPN login with LDAP credentials

CLI SSL VPN LDAP Configuration

  1. Configure SSL VPN to use LDAP groups 
    config user group
edit "VPN_Users"
    config group-authentication
        set authtimeout 480
        set sslvpn-portal "full-access"
    end
next
end
  2. Create SSL VPN policy with LDAP authentication 
    config firewall policy
edit 0
    set name "SSL_VPN_LDAP_Policy"
    set srcintf "ssl.root"
    set dstintf "internal"
    set srcaddr "SSLVPN_TUNNEL_ADDR1"
    set dstaddr "Internal_Networks"
    set action accept
    set schedule "always"
    set service "ALL"
    set users "VPN_Users"
    set groups "VPN_Users"
    set logtraffic all
    set nat enable
next
end
  3. Configure SSL VPN realm for LDAP 
    config vpn ssl settings
    config authentication-rule
        edit 1
            set groups "VPN_Users"
            set portal "full-access"
            set realm "ldap-realm"
        next
    end
end

Troubleshooting

Common LDAP authentication issues and their solutions.

Common Issues and Solutions

  1. LDAP server connection failed
    Screenshot: Connection failure indicators
    • Verify LDAP server IP address and port
    • Check network connectivity to domain controller
    • Ensure firewall allows LDAP traffic (389/636)
    • Verify service account credentials
  2. Authentication successful but no group membership
    Screenshot: Group membership issues
    • Check group search base DN
    • Verify group object filter syntax
    • Ensure user is member of configured groups
    • Check member attribute configuration
  3. SSL VPN login fails with LDAP user
    Screenshot: SSL VPN login failure
    • Verify user group has SSL VPN portal assigned
    • Check firewall policy allows group access
    • Ensure group authentication timeout is sufficient
    • Verify user account is enabled in AD

CLI Troubleshooting Commands

  1. Debug LDAP authentication 
    diagnose debug reset
diagnose debug enable
diagnose debug application fnbamd 255
  2. Test LDAP queries manually 
    diagnose test authserver ldap Company_AD john.doe password123 group-test
  3. Check authentication statistics 
    diagnose test authserver ldap Company_AD stat
  4. Monitor real-time authentication 
    execute log filter category 2
execute log display
  5. Verify group configuration 
    show user group VPN_Users
