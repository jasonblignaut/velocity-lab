Zabbix Monitoring Server Setup

Last updated: July 17, 2025

This guide provides step-by-step instructions for installing and configuring a Zabbix monitoring server for comprehensive network infrastructure monitoring.

Important

Ensure you have administrator access to your server and that ports 80, 443, and 10051 are available for Zabbix services.

Overview

Zabbix is an enterprise-class monitoring solution that provides real-time monitoring of servers, networks, applications, and services. This tutorial covers the complete setup process.

Prerequisites

  • Ubuntu 20.04 LTS or CentOS 8 server
  • Minimum 4GB RAM, 2 CPU cores
  • At least 20GB free disk space
  • Root or sudo access
  • Internet connectivity for package installation

CLI Installation Process

  1. Download and install Zabbix repository 
    # For Ubuntu 20.04
wget https://repo.zabbix.com/zabbix/6.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_6.0-1+ubuntu20.04_all.deb
sudo dpkg -i zabbix-release_6.0-1+ubuntu20.04_all.deb
sudo apt update
  2. Install Zabbix server, web frontend, and agent 
    sudo apt install zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-sql-scripts zabbix-agent
  3. Install and configure MySQL database 
    sudo apt install mysql-server
sudo mysql_secure_installation
sudo mysql -uroot -p
  4. Create Zabbix database and user 
    mysql> create database zabbix character set utf8mb4 collate utf8mb4_bin;
mysql> create user zabbix@localhost identified by 'password';
mysql> grant all privileges on zabbix.* to zabbix@localhost;
mysql> quit;
  5. Import initial schema and data 
    sudo zcat /usr/share/doc/zabbix-sql-scripts/mysql/server.sql.gz | mysql -uzabbix -p zabbix

Configuration File Setup

  1. Edit the Zabbix server configuration file 
    sudo nano /etc/zabbix/zabbix_server.conf
  2. Configure database connection 
    DBHost=localhost
DBName=zabbix
DBUser=zabbix
DBPassword=your_password
  3. Configure PHP settings for Zabbix frontend 
    sudo nano /etc/zabbix/apache.conf
  4. Set timezone in PHP configuration 
    php_value date.timezone Europe/London
  5. Start and enable Zabbix services 
    sudo systemctl restart zabbix-server zabbix-agent apache2
sudo systemctl enable zabbix-server zabbix-agent apache2

CLI-based Monitoring Setup

  1. Install Zabbix agent on target hosts 
    sudo apt install zabbix-agent
  2. Configure agent to connect to server 
    sudo nano /etc/zabbix/zabbix_agentd.conf
  3. Set server IP address 
    Server=192.168.1.100
ServerActive=192.168.1.100
Hostname=target-host
  4. Start and enable agent service 
    sudo systemctl restart zabbix-agent
sudo systemctl enable zabbix-agent
  5. Test connectivity from server 
    zabbix_get -s 192.168.1.101 -k system.cpu.load[all,avg1]

Troubleshooting

Common Issues

If you encounter database connection errors, verify that MySQL is running and the zabbix user has proper permissions.

Database Connection Issues

Check the Zabbix frontend for database connection status indicators and error messages in the system information page.

Screenshot: System information page showing database status

Check service status and logs for troubleshooting:

sudo systemctl status zabbix-server
sudo tail -f /var/log/zabbix/zabbix_server.log
sudo mysql -uzabbix -p zabbix -e "SELECT 1"
