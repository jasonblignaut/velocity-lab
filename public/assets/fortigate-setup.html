FortiGate Initial Setup

Last updated: July 17, 2025

This comprehensive guide covers the initial setup and configuration of FortiGate devices from first boot through basic security configuration.

Important

Always perform initial setup in a secure environment. Change default passwords immediately and ensure firmware is up to date.

Overview

FortiGate initial setup involves configuring basic network settings, security policies, and system parameters to prepare the device for production deployment. This process establishes the foundation for all subsequent configuration.

Setup Process Overview

  • Physical Setup: Hardware installation and connection
  • First Boot: Initial system startup and access
  • Basic Configuration: Network interfaces and system settings
  • Security Setup: Admin accounts and basic policies
  • System Configuration: Time, DNS, and logging
  • Verification: Testing and validation

Prerequisites

  • FortiGate device with power adapter
  • Ethernet cables for WAN and LAN connections
  • Computer with web browser for management
  • Console cable for CLI access (optional)
  • Network configuration plan (IP addresses, subnets)
  • Internet connection for license registration

CLI Initial Configuration

  1. Configure system global settings 
    config system global
    set hostname "FG-Branch-01"
    set timezone "GMT-5:00"
    set gui-language english
    set gui-theme mariner
    set admintimeout 60
    set admin-sport 443
    set admin-ssh-port 22
    set alias "Branch Office FortiGate"
end
  2. Change administrator password 
    config system admin
    edit "admin"
        set password "SecureAdminPassword123!"
        set gui-ignore-release-overview-version "7.0.0"
    next
end
  3. Configure DNS servers 
    config system dns
    set primary 8.8.8.8
    set secondary 8.8.4.4
end
  4. Configure NTP servers 
    config system ntp
    set ntpsync enable
    set type fortiguard
    set server-mode disable
end
  5. Set management interface access 
    config system interface
    edit "internal"
        set allowaccess ping https ssh
        set ip 192.168.1.99 255.255.255.0
    next
end

CLI Network Configuration

  1. Configure WAN interface 
    config system interface
    edit "port1"
        set vdom "root"
        set mode dhcp
        set allowaccess ping https ssh
        set type physical
        set alias "WAN"
        set role wan
        set snmp-index 1
    next
end
  2. Configure internal interface 
    config system interface
    edit "internal"
        set vdom "root"
        set ip 192.168.1.1 255.255.255.0
        set allowaccess ping https ssh telnet fgfm
        set type physical
        set alias "LAN"
        set role lan
        set snmp-index 2
    next
end
  3. Configure DMZ interface (if needed) 
    config system interface
    edit "port3"
        set vdom "root"
        set ip 192.168.2.1 255.255.255.0
        set allowaccess ping https ssh
        set type physical
        set alias "DMZ"
        set role dmz
        set snmp-index 3
    next
end
  4. Configure DHCP server for internal network 
    config system dhcp server
    edit 1
        set dns-service default
        set default-gateway 192.168.1.1
        set netmask 255.255.255.0
        set interface "internal"
        config ip-range
            edit 1
                set start-ip 192.168.1.100
                set end-ip 192.168.1.200
            next
        end
    next
end

Security Setup

Configure basic security settings including firewall policies and security profiles.

Warning

Default security policies may be very permissive. Review and adjust policies according to your security requirements.

Security Configuration via GUI

  1. Navigate to Policy & Objects > Firewall Policy
    Screenshot: Firewall Policy page
  2. Review default policies
    Screenshot: Default firewall policies
  3. Create basic security policy
    Screenshot: Create security policy form
    Setting Description Example
    Name Policy name Internal_to_Internet
    Source Interface Incoming interface internal
    Destination Interface Outgoing interface port1
    Source Address Source network all
    Destination Address Destination network all
    Service Allowed services ALL
    Action Policy action ACCEPT
    NAT Enable NAT ON
  4. Configure security profiles
    Screenshot: Security profiles configuration
  5. Enable logging
    Screenshot: Logging configuration

CLI Security Configuration

  1. Create basic firewall policy 
    config firewall policy
    edit 1
        set name "Internal_to_Internet"
        set srcintf "internal"
        set dstintf "port1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set nat enable
        set logtraffic all
        set comments "Allow internal users to access internet"
    next
end
  2. Create address objects 
    config firewall address
    edit "Internal_Network"
        set subnet 192.168.1.0 255.255.255.0
        set comment "Internal LAN network"
    next
    edit "DMZ_Network"
        set subnet 192.168.2.0 255.255.255.0
        set comment "DMZ network"
    next
end
  3. Configure security profiles 
    config firewall policy
    edit 1
        set av-profile "default"
        set ips-sensor "default"
        set application-list "default"
        set ssl-ssh-profile "certificate-inspection"
    next
end
  4. Configure admin access restrictions 
    config system admin
    edit "admin"
        set trusthost1 192.168.1.0 255.255.255.0
        set accprofile "super_admin"
        set vdom "root"
        set gui-ignore-release-overview-version "7.0.0"
    next
end

Configure system services, logging, and monitoring features.

System Services Configuration

  1. Navigate to System > Feature Visibility
    Screenshot: Feature Visibility page
  2. Enable required features
    Screenshot: Feature selection
  3. Configure logging settings
    Screenshot: Log settings configuration
  4. Set up SNMP (if required)
    Screenshot: SNMP configuration
  5. Configure backup settings
    Screenshot: Backup configuration

CLI System Configuration

  1. Configure logging settings 
    config log setting
    set status enable
    set local-in-policy enable
    set local-in-deny-unicast enable
    set local-out enable
    set daemon enable
end
  2. Configure SNMP (if required) 
    config system snmp sysinfo
    set status enable
    set description "FortiGate Branch Office"
    set contact-info "IT Support: support@company.com"
    set location "Branch Office"
end

config system snmp community
    edit 1
        set name "public"
        set query-v1-status enable
        set query-v2c-status enable
        set trap-v1-status enable
        set trap-v2c-status enable
        config hosts
            edit 1
                set ip 192.168.1.10 255.255.255.255
            next
        end
    next
end
  3. Configure FortiGuard services 
    config system fortiguard
    set antispam-force-off disable
    set antispam-cache enable
    set antispam-expiration 1618300800
    set outbreak-prevention-force-off disable
    set outbreak-prevention-cache enable
    set outbreak-prevention-expiration 1618300800
    set webfilter-force-off disable
    set webfilter-cache enable
    set webfilter-expiration 1618300800
    set sdns-server-ip "208.91.112.220"
    set sdns-server-port 53
    set anycast-sdns-server-ip "208.91.112.220"
    set anycast-sdns-server-port 53
    set sdns-options include-question
    set source-ip 0.0.0.0
    set source-ip6 ::
    set auto-join-forticloud enable
    set update-server-location automatic
end
  4. Configure firmware auto-update 
    config system autoupdate schedule
    set status enable
    set frequency every
    set time 02:00
    set day sunday
end
