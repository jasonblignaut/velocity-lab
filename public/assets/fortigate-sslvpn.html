FortiGate SSL VPN Configuration

Last updated: July 17, 2025

This comprehensive guide covers configuring SSL VPN on FortiGate devices to provide secure remote access for users connecting to corporate networks.

Important

SSL VPN requires a valid SSL certificate and proper DNS configuration. Ensure TCP port 443 is accessible from the internet.

Overview

FortiGate SSL VPN provides secure remote access using standard web browsers or the FortiClient application. It supports both web-based access and tunnel mode for full network connectivity.

SSL VPN Components

  • SSL VPN Portal: Web-based interface for user access
  • User Groups: Define access permissions and resources
  • Realms: Authentication domains for different user types
  • Policies: Control traffic flow for SSL VPN users
  • Client Certificates: Optional two-factor authentication

Prerequisites

  • FortiGate device with SSL VPN license
  • Valid SSL certificate (self-signed or CA-signed)
  • DNS name pointing to FortiGate's public IP
  • User authentication source (local, LDAP, or RADIUS)
  • Firewall policies allowing SSL VPN traffic

CLI SSL VPN Configuration

  1. Configure global SSL VPN settings 
    config vpn ssl settings
    set servercert "Fortinet_Factory"
    set tunnel-ip-pools "SSLVPN_TUNNEL_ADDR1"
    set tunnel-ipv6-pools "SSLVPN_TUNNEL_IPv6_ADDR1"
    set port 443
    set source-interface "internal"
    set source-address "all"
    set source-address6 "all"
    set default-portal "full-access"
end
  2. Configure IP pools for tunnel clients 
    config firewall address
    edit "SSLVPN_TUNNEL_ADDR1"
        set type iprange
        set start-ip 192.168.210.1
        set end-ip 192.168.210.50
        set comment "SSL VPN tunnel mode address pool"
    next
end
  3. Configure DNS servers for clients 
    config vpn ssl settings
    set dns-server1 192.168.1.10
    set dns-server2 8.8.8.8
    set wins-server1 192.168.1.10
end
  4. Enable SSL VPN on interface 
    config system interface
    edit "port1"
        set allowaccess ping https ssh
    next
end

CLI Portal Configuration

  1. Create SSL VPN portal 
    config vpn ssl web portal
    edit "employee-portal"
        set tunnel-mode enable
        set web-mode enable
        set ip-pools "SSLVPN_TUNNEL_ADDR1"
        set split-tunneling enable
        set theme blue
        set host-check custom
        set limit-user-logins disable
        set mac-addr-check disable
        set os-check disable
        set forticlient-download enable
        set skip-check-for-unsupported-browser disable
        set skip-check-for-browser disable
        set hide-sso-credential disable
        set auto-connect disable
        set keep-alive disable
        set save-password disable
        set use-sdwan disable
    next
end
  2. Configure split tunneling routes 
    config vpn ssl web portal
    edit "employee-portal"
        config split-tunneling-routing-address
            edit "Internal_Networks"
                set addr "192.168.1.0/24"
            next
            edit "Server_Network"
                set addr "192.168.10.0/24"
            next
        end
    next
end
  3. Add bookmarks for web access 
    config vpn ssl web portal
    edit "employee-portal"
        config bookmarks
            config group
                edit "Servers"
                    config bookmarks
                        edit "File Server"
                            set url "https://192.168.1.100"
                            set host "192.168.1.100"
                            set port 443
                        next
                        edit "Email Server"
                            set url "https://mail.company.com"
                            set host "mail.company.com"
                            set port 443
                        next
                    end
                next
            end
        end
    next
end

CLI User Group Configuration

  1. Create user group 
    config user group
    edit "SSL_VPN_Users"
        set member "john.doe" "jane.smith"
        config group-authentication
            set authtimeout 480
            set sslvpn-portal "employee-portal"
        end
    next
end
  2. Create local users 
    config user local
    edit "john.doe"
        set type password
        set passwd "SecurePassword123!"
        set status enable
        set email-to "john.doe@company.com"
        set sms-phone "555-0123"
        set sms-server "fortiguard"
        set two-factor disable
    next
    edit "jane.smith"
        set type password
        set passwd "SecurePassword456!"
        set status enable
        set email-to "jane.smith@company.com"
        set two-factor disable
    next
end
  3. Configure two-factor authentication 
    config user local
    edit "john.doe"
        set two-factor fortitoken
        set fortitoken "FTKMOB1234567890"
    next
end

Access Policies

Create firewall policies to control SSL VPN user access to internal resources.

Warning

SSL VPN policies should follow the principle of least privilege. Only grant access to required resources.

SSL VPN Access Policies

  1. Navigate to Policy & Objects > Firewall Policy
    Screenshot: Firewall Policy page
  2. Create policy for SSL VPN users
    Screenshot: SSL VPN policy creation
    Field Value Description
    Name SSL_VPN_to_Internal Policy identifier
    Incoming Interface ssl.root SSL VPN interface
    Outgoing Interface internal Internal network
    Source SSLVPN_TUNNEL_ADDR1 SSL VPN client pool
    Destination Internal_Networks Internal subnets
    Users SSL_VPN_Users User group
    Service HTTP, HTTPS, SSH Allowed services
    Action ACCEPT Allow traffic
  3. Configure authentication in policy
    Screenshot: Authentication settings in policy
  4. Enable logging for monitoring
    Screenshot: Logging configuration

CLI Access Policy Configuration

  1. Create SSL VPN access policy 
    config firewall policy
    edit 0
        set name "SSL_VPN_to_Internal"
        set srcintf "ssl.root"
        set dstintf "internal"
        set srcaddr "SSLVPN_TUNNEL_ADDR1"
        set dstaddr "Internal_Networks"
        set action accept
        set schedule "always"
        set service "HTTP" "HTTPS" "SSH" "PING"
        set users "SSL_VPN_Users"
        set logtraffic all
        set nat enable
    next
end
  2. Create address objects for internal networks 
    config firewall address
    edit "Internal_Networks"
        set subnet 192.168.1.0 255.255.255.0
        set comment "Internal LAN network"
    next
    edit "Server_Network"
        set subnet 192.168.10.0 255.255.255.0
        set comment "Server network"
    next
end
  3. Create service objects for specific applications 
    config firewall service custom
    edit "RDP"
        set tcp-portrange 3389
        set comment "Remote Desktop Protocol"
    next
    edit "File_Server"
        set tcp-portrange 445
        set comment "SMB file sharing"
    next
end

Troubleshooting

Common SSL VPN issues and their solutions.

Common Issues and Solutions

  1. Cannot access SSL VPN portal
    Screenshot: Connection error indicators
    • Check firewall allows port 443 access
    • Verify DNS resolution of VPN server
    • Ensure SSL certificate is valid
    • Check FortiGate system time
  2. Authentication failures
    Screenshot: Authentication error messages
    • Verify user credentials are correct
    • Check user account is enabled
    • Ensure user is in correct group
    • Verify LDAP/RADIUS connectivity
  3. Tunnel connects but no network access
    Screenshot: Tunnel status indicators
    • Check firewall policies allow SSL VPN traffic
    • Verify routing configuration
    • Ensure IP pool is correctly configured
    • Check DNS settings for clients

CLI Troubleshooting Commands

  1. Check SSL VPN status 
    get vpn ssl monitor
diagnose vpn ssl list
  2. Monitor authentication attempts 
    diagnose debug enable
diagnose debug application sslvpn -1
  3. Check user sessions 
    get vpn ssl stats
diagnose vpn ssl list
  4. Test DNS resolution 
    diagnose test application dnsproxy 6
execute nslookup vpn.company.com
  5. Monitor real-time logs 
    execute log filter category 2
execute log display
