FortiGate Firewall Policy Creation and Management

Last updated: July 17, 2025

This guide explains how to create and manage FortiGate firewall policies using the Web Interface or CLI with step-by-step instructions.

Important

Ensure you have administrator access to the FortiGate device and that FortiOS is running version 6.4 or later for best compatibility.

Prerequisites

  • Administrator access to FortiGate device
  • FortiOS version 6.4 or later
  • Understanding of network zones and interfaces
  • Basic knowledge of firewall security concepts

Create Firewall Policy

  • FortiGate Web Interface
  • CLI
  1. Navigate to Policy & Objects > Firewall Policy in the FortiGate GUI
    Screenshot: FortiGate main menu showing Policy & Objects navigation
  2. Click "Create New" to start creating a new firewall policy
    Screenshot: Firewall Policy list with Create New button highlighted
  3. Configure the basic policy settings:
    Setting Action
    Name Enter a descriptive name for the policy
    Incoming Interface Select the source interface/zone
    Outgoing Interface Select the destination interface/zone
    Source Define source addresses or address groups
    Destination Define destination addresses or address groups
    Service Select the services/ports to allow
    Action Choose ACCEPT, DENY, or IPSEC
    Screenshot: New Firewall Policy configuration form
  4. Click "OK" to create the policy
    Screenshot: Completed policy creation form with OK button
  1. Connect to FortiGate CLI and enter configuration mode 
    
config firewall policy
  2. Create a new policy with a unique policy ID 
    
edit 1
  3. Configure the policy parameters 
    
set name "Allow_LAN_to_WAN"
set srcintf "internal"
set dstintf "wan1"
set srcaddr "all"
set dstaddr "all"
set service "ALL"
set action accept
set schedule "always"
  4. Save the configuration 
    
next
end

Configure Security Features

  • FortiGate Web Interface
  • CLI
  1. Edit the created policy to add security features
    Screenshot: Policy list with edit option highlighted
  2. Navigate to the Security Profiles section
    Screenshot: Policy edit form showing Security Profiles tab
  3. Enable and configure security profiles:
    Security Profile Configuration
    Antivirus Enable default or custom antivirus profile
    Web Filtering Configure web filtering categories
    Application Control Set application control policies
    IPS Enable intrusion prevention system
    Screenshot: Security Profiles configuration section
  4. Apply the changes and save the policy
    Screenshot: Policy save confirmation
  1. Edit the existing policy to add security profiles 
    
config firewall policy
edit 1
  2. Configure antivirus profile 
    
set av-profile "default"
  3. Configure web filtering 
    
set webfilter-profile "default"
  4. Configure application control and IPS 
    
set application-list "default"
set ips-sensor "default"
next
end

Verify Policy Configuration

  • FortiGate Web Interface
  • CLI
  1. Review the policy list to confirm creation
    Screenshot: Policy list showing newly created policy
  2. Check policy hit counts in the statistics column
    Screenshot: Policy statistics showing hit counts
  3. Test connectivity through the policy using network tools
    Screenshot: Policy monitoring or log viewer
  1. Display all firewall policies 
    
show firewall policy
  2. Check specific policy details 
    
show firewall policy 1
  3. Monitor policy hit counts 
    
get firewall policy
