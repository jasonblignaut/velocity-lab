FortiGate SD-WAN Configuration

Last updated: July 17, 2025

This comprehensive guide covers configuring Software-Defined WAN (SD-WAN) on FortiGate devices to optimize network performance, provide redundancy, and improve application delivery.

Important

SD-WAN configuration requires FortiOS 5.4 or later. Ensure you have multiple WAN connections configured before proceeding with SD-WAN setup.

Overview

FortiGate SD-WAN provides intelligent path selection, load balancing, and failover capabilities for multiple WAN connections. It uses real-time application and link quality metrics to optimize traffic routing and ensure optimal user experience.

Key Benefits

  • Automatic failover and load balancing
  • Application-aware routing
  • Real-time link quality monitoring
  • Bandwidth optimization
  • Centralized management and visibility

Prerequisites

  • FortiGate device with FortiOS 5.4 or later
  • Multiple WAN connections (minimum 2)
  • Configured WAN interfaces with IP addresses
  • Static routes or default routes configured
  • Administrator access to FortiGate management interface

Interface Setup

Before configuring SD-WAN, ensure your WAN interfaces are properly configured and operational.

Interface Configuration via GUI

  1. Navigate to Network > Interfaces
    Screenshot: Network Interfaces page showing WAN interfaces
  2. Edit each WAN interface (port2, port3, etc.)
    Screenshot: Interface configuration form
  3. Configure IP addressing and administrative settings
    Setting Description Example
    Role Interface role WAN
    Administrative Status Interface state Up
    IP/Network Mask Interface IP address 192.168.100.2/24
    Administrative Distance Route preference 10
  4. Configure DNS settings for each interface
    Screenshot: DNS configuration section

CLI Interface Configuration

  1. Configure WAN interface 1 (port2) 
    config system interface
edit "port2"
    set vdom "root"
    set ip 192.168.100.2 255.255.255.0
    set allowaccess ping https ssh
    set type physical
    set role wan
    set interface port2
    set distance 10
next
  2. Configure WAN interface 2 (port3) 
    edit "port3"
    set vdom "root"
    set ip 192.168.200.2 255.255.255.0
    set allowaccess ping https ssh
    set type physical
    set role wan
    set interface port3
    set distance 20
next
end
  3. Configure static routes 
    config router static
edit 1
    set gateway 192.168.100.1
    set device "port2"
    set distance 10
next
edit 2
    set gateway 192.168.200.1
    set device "port3"
    set distance 20
next
end

SD-WAN Zones

SD-WAN zones group interfaces and define how traffic is distributed across multiple WAN links.

Creating SD-WAN Zones

  1. Navigate to Network > SD-WAN
    Screenshot: SD-WAN configuration page
  2. Click "Enable SD-WAN"
    Screenshot: Enable SD-WAN button
  3. Add WAN interfaces to SD-WAN zone
    Screenshot: Add interface to SD-WAN zone
  4. Configure interface priorities and weights
    Interface Priority Weight Description
    port2 1 100 Primary WAN
    port3 2 50 Secondary WAN
  5. Configure gateway and detection settings
    Screenshot: Gateway and detection configuration

CLI SD-WAN Zone Configuration

  1. Enable SD-WAN and add interfaces 
    config system sd-wan
    set status enable
    config members
        edit 1
            set interface "port2"
            set zone "virtual-wan-link"
            set priority 1
            set weight 100
            set gateway 192.168.100.1
        next
        edit 2
            set interface "port3"
            set zone "virtual-wan-link"
            set priority 2
            set weight 50
            set gateway 192.168.200.1
        next
    end
  2. Configure health check settings 
        config health-check
        edit "Default_Office_365"
            set server "outlook.office365.com"
            set protocol ping
            set port 443
            set interval 1000
            set probe-timeout 1000
            set failtime 3
            set recoverytime 3
            config members
                edit 1
                    set seq-num 1
                next
                edit 2
                    set seq-num 2
                next
            end
        next
    end
  3. Configure basic load balancing rule 
        config service
        edit 1
            set name "Internet_Service"
            set mode load-balance
            set dst "all"
            set src "all"
            config sla
                edit "Default_Office_365"
                    set id 1
                next
            end
            set priority-members 1 2
        next
    end
end

Performance SLA

Performance SLA monitoring ensures that traffic is routed over links that meet specific quality requirements.

Configuring Performance SLA

  1. In SD-WAN settings, navigate to Performance SLA
    Screenshot: Performance SLA configuration page
  2. Create new SLA profile
    Screenshot: Create new SLA profile form
  3. Configure SLA parameters
    Parameter Description Example Value
    Latency Threshold Maximum acceptable latency 150ms
    Jitter Threshold Maximum acceptable jitter 50ms
    Packet Loss Maximum acceptable packet loss 2%
    Probe Interval How often to test the link 10 seconds
  4. Configure target servers for monitoring
    Screenshot: Target server configuration

CLI Performance SLA Configuration

  1. Configure health check for critical applications 
    config system sd-wan
    config health-check
        edit "VoIP_SLA"
            set server "voip.company.com"
            set protocol ping
            set port 5060
            set interval 5000
            set probe-timeout 1000
            set failtime 3
            set recoverytime 3
            set sla-fail-log-period 10
            set sla-pass-log-period 10
            config sla
                edit 1
                    set link-cost-factor latency jitter packet-loss
                    set latency-threshold 100
                    set jitter-threshold 20
                    set packetloss-threshold 1
                next
            end
        next
    end
  2. Configure health check for general internet 
            edit "Internet_SLA"
            set server "8.8.8.8"
            set protocol ping
            set interval 10000
            set probe-timeout 2000
            set failtime 5
            set recoverytime 5
            config sla
                edit 1
                    set link-cost-factor latency packet-loss
                    set latency-threshold 200
                    set packetloss-threshold 3
                next
            end
        next
    end
end

Load Balancing

Configure load balancing algorithms to distribute traffic across multiple WAN links based on various criteria.

Warning

Load balancing algorithms can affect application performance. Test thoroughly with your specific traffic patterns.

Load Balancing Configuration

  1. Navigate to SD-WAN Rules section
    Screenshot: SD-WAN Rules configuration
  2. Create new SD-WAN rule
    Screenshot: Create new SD-WAN rule form
  3. Configure load balancing algorithm
    Algorithm Description Best For
    Source IP Based on source IP hash Session persistence
    Weighted Based on interface weight Bandwidth proportional
    Spillover Primary with failover Active-passive setup
    Volume Based on traffic volume Equal distribution
  4. Configure application-specific rules
    Screenshot: Application-specific rule configuration

CLI Load Balancing Configuration

  1. Configure weighted load balancing for general traffic 
    config system sd-wan
    config service
        edit 1
            set name "General_Internet"
            set mode load-balance
            set dst "all"
            set src "all"
            set priority-members 1 2
            set tie-break farthest
            set use-shortcut-sla disable
        next
    end
  2. Configure priority-based routing for VoIP 
            edit 2
            set name "VoIP_Traffic"
            set mode priority
            set dst "all"
            set src "all"
            set internet-service-app-ctrl-group "VoIP.Audio"
            config sla
                edit "VoIP_SLA"
                    set id 1
                next
            end
            set priority-members 1 2
        next
  3. Configure spillover for critical applications 
            edit 3
            set name "Critical_Apps"
            set mode manual
            set dst "all"
            set src "all"
            set internet-service-app-ctrl-group "Business.Applications"
            config sla
                edit "Internet_SLA"
                    set id 1
                next
            end
            set priority-members 1
        next
    end
end

Troubleshooting

Common SD-WAN issues and their solutions.

Common Issues and Solutions

  1. Interface not showing in SD-WAN zone
    Screenshot: Interface troubleshooting steps
    • Check interface role is set to WAN
    • Verify interface is administratively up
    • Confirm IP configuration is correct
  2. Health check failing
    Screenshot: Health check failure indicators
    • Verify target server is reachable
    • Check firewall policies allow health check traffic
    • Adjust timeout and interval settings
  3. Traffic not load balancing
    Screenshot: Traffic distribution analysis
    • Review SD-WAN rule configuration
    • Check interface weights and priorities
    • Verify SLA requirements are met

CLI Troubleshooting Commands

  1. Debug SD-WAN operations 
    diagnose debug enable
diagnose debug application sdwan -1
diagnose debug enable
  2. Check routing table 
    get router info routing-table all
  3. Test connectivity 
    execute ping-options source 192.168.100.2
execute ping 8.8.8.8
  4. Monitor session table 
    diagnose sys session list
  5. Check interface statistics 
    diagnose hardware deviceinfo nic port2
